### 完整流程演绎

```mermaid placement="bottom-right"
sequenceDiagram
    autonumber
    
    box rgb(252, 228, 236) "External"
    actor User as Telegram / Feishu App
    end

    box rgb(232, 234, 246) "Access Agnostic Layer"
    participant OmniGW as Omnichannel Gateway
    end

    box rgb(227, 242, 253) "Agentic-Layer"
    participant AAIHF as AAIHF (adk-go Reasoner)
    participant ACRF as ACRF (Skill Registry)
    participant IGW as A-IGW (Translator)
    end
    
    box rgb(255, 248, 225) "Legacy 5G Core [free5gc]"
    participant AMF as AMF
    participant SMF as SMF
    participant NEF as NEF
    end

    %% Stage 1: Intent
    Note over User, AAIHF: Stage 1: Intent
    User->>OmniGW: Chat: "Wake up fleet for firmware update"
    OmniGW->>AAIHF: POST /intent {prompt, user_id}
    Note over AAIHF: LLM processes natural language<br/>Determines required capability
    
    %% Stage 2: Skill
    Note over AAIHF, ACRF: Stage 2: Skill
    AAIHF->>ACRF: SearchSkill("mcp://skill/device/fleet-update")
    ACRF-->>AAIHF: Return Profile (AgenticService-URI: http://igw:8081/invoke)
    
    %% Stage 3: Service Directive
    Note over AAIHF, NEF: Stage 3: Service Directive
    AAIHF->>IGW: ExecuteSkill (POST /invoke)
    
    Note over IGW: A-IGW acts as Universal Translator<br/>Maps 1 Skill to 3 Core APIs
    
    IGW->>AMF: 1. Namf_MT_EnableUEReachability
    AMF-->>IGW: 200 OK
    
    IGW->>SMF: 2. Nsmf_PDUSession_UpdateSMContext
    SMF-->>IGW: 200 OK
    
    IGW->>NEF: 3. Nnef_AFSessionWithQoS_Create
    NEF-->>IGW: 201 Created
    
    %% Return Path
    Note over IGW, User: Confirmation & Feedback
    IGW-->>AAIHF: Success (Translation Sequence Completed)
    Note over AAIHF: LLM generates human-friendly summary
    AAIHF-->>OmniGW: {response: "Fleet successfully woken up..."}
    OmniGW-->>User: Chat: "Fleet successfully woken up and QoS provisioned."
```

1. ACRF实现简化：其注册表用一个简单的`map[string]SkillProfile`，只能做字符串匹配，没法实现语义搜索。
2. AAIHSF实现简化：第3步SearchSkill时，直接使用`mcp://skill/device/fleet-update`。正常来说，应该携带required-skill=`ensure-device-reachability` or `mcp://skill/wake-up-devices`

## 运行视图

#### Google提案

<Frame>
  ![Image](/images/image-2.png)
</Frame>

- Step 1, NF -> ACRF/NRF: NF sends message to ACRF/NRF. The message includes Nacrf_SkillManagement_Register Request/Nnrf_NFManagement_NFRegister Request | (Node_Type='NF', agenticSkillSet).
- Step 2, ACRF/NRF -> NF: ACRF/NRF sends message to NF. The message includes Nacrf_SkillManagement_Register Response/Nnrf_NFManagement_NFRegister Response | (200 OK).
- Step 3, AAIHF (Core AI Agent): AAIHF (Core AI Agent) perform internal Intent ID(s).
- Step 4, AAIHF (Core AI Agent) -> ACRF/NRF: AAIHF (Core AI Agent) sends message to ACRF/NRF. The message includes Nacrf_SkillDiscovery_Request/Nnrf_NFDiscovery_Request | (required-skill).
- Step 5, ACRF/NRF -> AAIHF (Core AI Agent): ACRF/NRF sends message to AAIHF (Core AI Agent). The message includes Nacrf_SkillDiscovery_Response/Nnrf_NFDiscovery_Response | (List of Agentic_Target_Profile(s)).
